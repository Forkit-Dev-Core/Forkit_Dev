# Code of Conduct

## Our standard

Forkit Dev Core is a technical open source project. We expect discussions to stay respectful, direct, and constructive.

Examples of expected behavior:

- focus on the code, docs, reproduction, or design tradeoff
- assume good intent and ask clarifying questions before escalating
- give concrete technical feedback instead of personal criticism
- be transparent about uncertainty, blockers, and validation gaps

Examples of unacceptable behavior:

- harassment, discrimination, or intimidation
- personal attacks or abusive language
- doxxing or sharing private information without permission
- bad-faith disruption, spam, or repeated off-topic escalation

## Scope

This code of conduct applies to repository issues, pull requests, discussions, and other project spaces.

## Reporting

If you encounter behavior that violates this code of conduct, contact the maintainers through GitHub with enough context to investigate. Please avoid public pile-ons; a concise report is more useful than a long argument thread.

## Enforcement

Maintainers may edit, remove, or lock content that violates this code of conduct and may restrict participation for repeated or severe violations.
